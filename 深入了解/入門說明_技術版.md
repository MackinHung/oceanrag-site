# OceanRAG — 入門說明（技術版）

---

## 1. 快速介紹

OceanRAG 是一套企業內部 RAG（Retrieval-Augmented Generation）系統，設計目標是在零外部依賴的前提下，提供生產級的知識問答、權限控制與安全稽核能力。

**技術定位：**
- 向量檢索：Jina v5（1024-dim），存於 LanceDB
- LLM 生成：Groq（可熱切換，不鎖定 vendor）
- 關聯式資料：SQLite（含 trigger 保護的稽核日誌）
- 無需 Redis / PostgreSQL / Elasticsearch

**架構：** 四個獨立 Tier（T1 前端 / T2 RAG 引擎 / T3 文件處理 / T4 平台管理），層間以 `X-Service-Token` 通訊，限流 30 req/min/IP。

**認證：** JWT RS256（非對稱），httpOnly + SameSite=Strict cookie，T2 不持有私鑰，僅從 proxy 轉發的 JWT payload 提取用戶身份。

---

## 2. 功能特色

### 智慧問答引擎

SSE 即時串流回答。支援多輪對話（session_id 追蹤），查詢前執行意圖分類與查詢改寫，再進入向量搜尋流程。

核心技術：Jina v5 embedding、Groq LLM、多輪上下文記憶。

### 信心分數查核（NLI Pipeline）

回答生成後，對每個 claim 執行 NLI 推論（Erlangshen-Roberta-330M-NLI），分類為 Entail / Neutral / Contradict，輸出 PASS / WARN / BLOCK 三級結果。財務類 claim 走獨立分類路由。

### 文件 ETL Pipeline

8 階段自動化處理：解析 → 清洗 → 切塊 → 摘要 → 向量化 → 品質檢測（5 維度，滿分 100，60 分以下警示）→ 入庫 → 通知。支援 PDF / DOCX / XLSX，含 magic number 格式驗證。

### 三層門控（無關內容截斷）

- **A-layer**：意圖分類，非知識庫查詢提早攔截
- **B-layer**：向量閾值 0.4 + reranker 閾值 0.6，雙重把關，不達標不送 LLM
- **C-layer**：NLI 信心評分，BLOCK 級別直接攔截回答

實測約 30%～50% 查詢在 B-layer 被攔截，有效降低 LLM 呼叫成本。

### 四層授權架構

每個請求依序通過：
1. JWT Cookie + Session 驗證（401）
2. 角色檢查：employee / dept_admin / master_admin（403）
3. 機密等級：用戶 CL >= 文件 CL（403）
4. 部門歸屬 + 功能權限矩陣（403）

RBAC pre-filter 在 LanceDB 向量搜尋前執行，`clearance_level` 強制轉型，`ValueError` if null，不可繞過。

### 其他功能模組

- **對話紀錄**：session_id 追蹤，upsert 機制，SSE done event 攜帶 session_id
- **用量統計**：三層 drill-down（部門 → 使用者 → 查詢），Chart.js，KPI 計算，CSV 匯出
- **AI 模型管理**：generation / embedding / reranker / routing 各自可切換，健康探測，熱切換不重啟
- **文件審閱**：PDF.js / mammoth.js 即時預覽，批量操作
- **風險監控**：風險分數 0–100（EXP_DECAY 加權），60 分警示 / 80 分嚴重警報，冷卻機制 4 小時
- **來源追溯**：citation 映射，PDF bounding box highlighting

## 3. 核心效果

### 與傳統 RAG 的差異

| 面向 | 傳統 RAG | OceanRAG |
|------|---------|---------|
| 查詢理解 | 單輪 keyword 搜尋 | 意圖識別 + 查詢改寫 + 多輪上下文 |
| 答案驗證 | 無 | NLI 逐 claim 比對，Entail/Neutral/Contradict |
| 幻覺防護 | 無 | 三層門控，BLOCK 直接攔截 |
| 文件整理 | 人工 | 全自動 8 階段 ETL + 品質分數 |
| 權限控制 | 全員可見 | RBAC pre-filter，向量搜尋前執行，不可繞過 |
| 部署依賴 | 需 5+ 外部服務 | SQLite + LanceDB，零外部依賴 |
| Token 安全 | 多用 localStorage（XSS 風險） | httpOnly cookie + JWT RSA-2048 |
| 稽核日誌 | 可刪除 | SQLite DELETE trigger 保護，SHA-256 雜湊鏈 |

### 成本結構

- LLM 成本：Groq 約為 GPT-4 的 1/10～1/20
- 門控節省：B-layer 攔截 30%～50% 無效呼叫
- Embedding：文件入庫時執行一次，查詢只做 query embedding

### 安全設計要點

- JWT RS256 非對稱加密，T2 不持有私鑰
- 角色或機密等級變更後立即撤銷所有 session（防 session fixation）
- 安全事件 SHA-256 雜湊鏈，即使 master_admin 也無法竄改
- 時序攻擊防護：帳號不存在時執行 dummy bcrypt
